HEROVERSE KIDS — BACKEND IMPLEMENTATION SPEC (Phase 1)
=======================================================
Read this entire file before writing any code.
Work only inside: apps/api/src/

PROJECT
-------
HeroVerse Kids is an AI-powered personalized storybook platform. Parents sign up,
create a hero (their child), build a supporting cast (family/pets), and generate
personalized illustrated stories. The child is always the hero.

IMPORTANT RULES
---------------
- Parents ONLY sign up. Children never have accounts.
- A child's photo is NEVER stored. It is processed to generate an AI avatar only.
- 1 Credit = 1 complete story generation.
- Relationships for supporting characters: father | mother | brother | sister |
  grandpa | grandma | friend | pet  (one character per relationship per user)


TECH STACK
----------
- NestJS 11 (TypeScript, strict mode)
- PostgreSQL via TypeORM
- Redis via ioredis (for BullMQ job queues later)
- JWT for auth (access + refresh tokens)
- bcrypt for password hashing
- class-validator + class-transformer for DTOs
- Cloudflare R2 (S3-compatible) for avatar + asset storage
- @nestjs/config for environment variables

ENDPOINTS (all @Public()):

POST /auth/register
  Body: { name: string, email: string, password: string, referralCode?: string }
  Validation:
    - name: minLength 2
    - email: isEmail
    - password: minLength 8
  Logic:
    1. Check email not already registered (throw 409 if exists)
    2. bcrypt.hash(password, 12)
    3. Create User with role='parent'
    4. If referralCode provided: store in user.referredBy (validate code exists — soft fail if not)
    5. Generate and return access + refresh tokens
  Response: { accessToken, refreshToken, user: { id, name, email, credits } }

POST /auth/login
  Body: { email: string, password: string }
  Logic:
    1. Find user by email (throw 401 if not found)
    2. bcrypt.compare (throw 401 if mismatch)
    3. Return tokens
  Response: { accessToken, refreshToken, user: { id, name, email, credits } }

POST /auth/refresh
  Guard: JwtRefreshGuard (reads Bearer token from Authorization header)
  Logic: Validate refresh token, issue new access token
  Response: { accessToken }

POST /auth/logout
  Guard: JwtAuthGuard
  Logic: (For now just return 200 — token blacklist comes later)

GET /auth/me
  Guard: JwtAuthGuard
  Response: { id, name, email, credits, createdAt }

JWT payload shape: { sub: userId, email }
JWT access secret from JWT_ACCESS_SECRET, expires JWT_ACCESS_EXPIRES
JWT refresh secret from JWT_REFRESH_SECRET, expires JWT_REFRESH_EXPIRES

All endpoints require JwtAuthGuard. Hero always belongs to req.user.

POST /heroes
  Body: { name, age, gender }
  Logic: Create hero for current user. Return hero.
  Note: A user can have multiple heroes (different children).

GET /heroes
  Logic: Return all heroes for current user.

GET /heroes/:id
  Logic: Return hero if it belongs to current user, else 404.

PATCH /heroes/:id
  Body: Partial<CreateHeroDto>
  Logic: Update hero fields. Return updated hero.

DELETE /heroes/:id
  Logic: Soft-check for associated stories before deleting. Return 204.

Supporting cast. One per relationship per user.

POST /characters
  Body: { relationship, name, description? }
  Logic:

GET /characters

GET /characters/:id
  Logic: Return character if it belongs to current user.

PATCH /characters/:id
  Body: { name?, description? }

DELETE /characters/:id
  Logic: Delete. Return 204.


─────────────────────────────────────────────────────────────────────────────
MODULE 5: StoriesModule  (src/stories/)
─────────────────────────────────────────────────────────────────────────────
Files: stories.module.ts, stories.controller.ts, stories.service.ts,
       story.entity.ts, dto/create-story.dto.ts

All endpoints require JwtAuthGuard.

POST /stories
  Body: { heroId, theme, castIds?: string[] }
  Validation:
    - heroId must exist and belong to current user
    - theme must be valid enum value
    - castIds: array of SupportingCharacter UUIDs belonging to current user (max 4)
    - User must have credits >= 1 (throw 402 with message "Not enough credits")
  Logic:
    1. Deduct 1 credit from user
    2. Log CreditTransaction { delta: -1, reason: 'story_generation', referenceId: story.id }
    3. Create story with status 'pending'
    4. (Queue a BullMQ job — stub this for now: just set status to 'pending' and return)
    5. Return created story

GET /stories
  Logic: Return all stories for current user (newest first). Include hero name and theme.

GET /stories/:id
  Logic: Return story if it belongs to current user. Include full pages array.

DELETE /stories/:id
  Logic: Delete story. Return 204. (Do not refund credits.)

─────────────────────────────────────────────────────────────────────────────
MODULE 7: UploadModule  (src/upload/)
─────────────────────────────────────────────────────────────────────────────
Files: upload.module.ts, upload.controller.ts, upload.service.ts

Handles photo upload → will trigger AI avatar generation (AI stub for now).

POST /upload/avatar
  Guard: JwtAuthGuard
  Body: multipart/form-data, field name: "photo", max size: 5MB, accept: image/jpeg, image/png, image/webp
  Logic:
    1. Validate file type and size (throw 400 if invalid)
    2. Generate a unique filename: `avatars/{userId}/{uuid}.webp`
    3. For now: upload the original file to R2 as-is (AI avatar generation is a future step)
    4. Return { avatarUrl: "https://..." }
  Note: This endpoint will be called from hero creation and character creation flows.
  The actual photo is NEVER returned or stored long-term — in production,
  it will be processed and discarded. For now we store it temporarily.

Use @aws-sdk/client-s3 with S3-compatible config for R2:
  endpoint: `https://${R2_ACCOUNT_ID}.r2.cloudflarestorage.com`
  region: 'auto'
  credentials: { accessKeyId: R2_ACCESS_KEY_ID, secretAccessKey: R2_SECRET_ACCESS_KEY }


GLOBAL SETUP
=============

src/main.ts:
  - app.useGlobalPipes(new ValidationPipe({ whitelist: true, transform: true }))
  - app.enableCors({ origin: ['http://localhost:3001', 'https://heroversekids.com'] })
  - app.setGlobalPrefix('api')
  - Listen on process.env.PORT || 3000
  - Swagger: DocumentBuilder with title 'HeroVerse Kids API', version '1.0'
    app.use('/api/docs', SwaggerModule) — @nestjs/swagger

src/common/filters/http-exception.filter.ts:
  Global exception filter that returns:
  { statusCode, message, error, timestamp, path }

src/common/interceptors/transform.interceptor.ts:
  Wrap all successful responses in: { success: true, data: <payload>, timestamp }
